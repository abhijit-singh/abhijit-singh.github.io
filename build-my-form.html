BuildMyForm

A simple & intuitive plugin to enable operators to create custom forms to record user details during or after bookings.

    UX, Visual

  • Platform

  • Team

Project Brief

Before this project was undertaken, an admin could create a form by choosing from a handful of preset fields. These fields can ask the end user for information once per booking or once per passenger, but only while they make a booking. The fields can also be configured to be mandatory.

We now needed to allow the admin to configure fields for the end user to fill either before, during, or even after a booking was made. The UI needed to be easy to setup such that the admin can use the same forms in multiple products without having to set them up again. Hence this was decided to be built as an external plugin which would allow the admin to build any kind of custom forms and link them to as many products as they want.

My Role

While I was involved in the research and conceptualization phase of the project, it was primarily driven by the product manager. Working closely with him, we came up with an initial spec document outlining the approach we needed to take for this project. After that I took over the project and created wireframes, mockups, a clickable prototype for it.

Wireframes

These were completed quickly enough as we'd already spent a good amount of time narrowing down the broad level approach. The UX to setup and add new fields was to be figured out and the various setting pertaining to each kind of field to be found and added. I came up with the idea of maintaining the commonly used settings for each kind of field in a smart master library and applying them whenever a field is added again in a similar context. The context was found based on the product type, form type and occurence of other fields with respect to a given field inside the form.

Mockups & Final Prototype

High fidelity mockups were finally created using Sketch & were converted into a clickable prototype using Invision. A master list for the kind of fields, settings pertaining to each field and the possible validations on it was also designed. One neat addition that I was happy to incorporate was using interactive and friendly validation messages, which weren't being used anywhere else in the system.
